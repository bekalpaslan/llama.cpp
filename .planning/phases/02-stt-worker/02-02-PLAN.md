**GREEN phase:**

Rewrite `worker-whisper/handler.py` (replacing the template skeleton) with:

**Module-level startup:**
- Read env vars: MODEL_SIZE (default "large-v3-turbo"), COMPUTE_TYPE (default "int8"), HF_TOKEN, HF_REPO_ID, MODEL_NAME
- If HF_REPO_ID is set, use `download_model(repo_id=HF_REPO_ID, token=HF_TOKEN or None, snapshot=True)` to download the model directory. Otherwise skip (model_size will be used directly by WhisperModel).
- Create `WhisperModel(MODEL_SIZE, device="cuda", compute_type=COMPUTE_TYPE)`. If HF_REPO_ID was used and returned a path, pass that as model_size_or_path instead.
- Try to load diarization pipeline: if HF_TOKEN is set, try `from whisperx.diarize import DiarizationPipeline` and create `diarize_pipeline = DiarizationPipeline(use_auth_token=HF_TOKEN, device="cuda")`. Catch any exception, log warning, set `diarize_pipeline = None`.
- Set `PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True` via os.environ if not already set.
- Import transcribe_audio, format functions, audio_utils functions.

**handler(job) function:**
- Extract job_input from job["input"]
- Route to `_process_batch` if "audio_urls" in job_input, else `_process_single`
- Return result

**_process_single(job_input) function:**
- Resolve audio input via resolve_audio_input(job_input)
- Extract parameters: language, task (default "transcribe"), word_timestamps (default True), output_format (default "json"), diarize (default False), beam_size (default 5), temperature (default 0.0), min_speakers, max_speakers
- Call transcribe_audio(model, audio_path, ...)
- If diarize is True, call `_process_diarization(audio_path, result["segments"], job_input)`
- Apply output format: if "text" -> add result["text"], if "srt" -> add result["srt"], if "vtt" -> add result["vtt"]
- In finally block: cleanup_audio, torch.cuda.empty_cache()
- Return result. Catch ValueError -> {"error": str(e)}, catch Exception -> {"error": f"Transcription failed: {str(e)}"}

**_process_batch(job_input) function:**
- Extract audio_urls list
- Extract shared params (language, task, word_timestamps, output_format, diarize, etc.)
- Loop through audio_urls, for each create a synthetic single-input dict with audio_url + shared params, call _process_single
- Return {"results": [...]}

**_process_diarization(audio_path, segments, job_input) function:**
- If diarize_pipeline is None: return dict with "warning" explaining HF_TOKEN requirement (mention HF_TOKEN, pyannote/speaker-diarization-3.1, user agreement) and the unmodified segments
- Otherwise: use whisperx.load_audio, run diarize_pipeline with min_speakers/max_speakers, call whisperx.assign_word_speakers, return result with speaker-labeled segments

**At module bottom:**
- `runpod.serverless.start({"handler": handler})`

Run tests -- all must PASS.
