<objective>
Create a standalone voice-mixing module that implements weighted Kokoro voice blending (TTS-01), along with patched versions of the TTS worker files ready for drop-in integration.

Purpose: The existing TTS worker at `tts-worker/` supports single voices only. This module adds the ability to blend 2-5 Kokoro voices with arbitrary weights (e.g., `af_heart:70,af_bella:30`), producing a weighted-sum voice tensor that generates audibly distinct blended speech.

Output: A `tts-voice-mixing/` directory containing the core blend module with tests, patched engine/handler files, updated test cases, and integration instructions.
</objective>

  <action>
    Create `tts-voice-mixing/voice_blend.py` with three public functions:

    1. `parse_voice_spec(voice_spec: str) -> list[tuple[str, float]]`
       - Split on comma. For each part, split on colon to get name and optional weight.
       - Default weight is 1.0 when no colon present.
       - Normalize all weights to sum to 1.0.
       - Validate: all weights must be positive numbers, max 5 voices, no empty names.
       - Raise ValueError with descriptive message on invalid input.

    2. `validate_blend_request(voice: str, known_voices: dict, engine: str) -> str | None`
       - If voice contains no comma AND no colon-with-comma, return None (single voice, no blend validation needed).
       - If engine != "kokoro", return error: voice blending only supported for Kokoro.
       - Parse the voice spec. If any voice name not in known_voices, return error naming the unknown voice.
       - If voice names have different first-character language prefixes, return error about cross-language blending.
       - Return None on success.

    3. `blend_voices(pipeline, voice_parts: list[tuple[str, float]]) -> torch.FloatTensor`
       - If single voice, call `pipeline.load_voice(name)` and return directly.
       - For multiple voices, load each tensor via `pipeline.load_voice(name)`, compute `sum(tensor * weight for tensor, weight in parts)`, return blended tensor.
       - Import torch only inside this function (lazy import) to keep the module importable without torch for testing.
